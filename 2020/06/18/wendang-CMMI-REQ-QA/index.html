CMMI项目管理QA

发表于 2020-06-18 | 更新于 2020-06-18 | 分类于 文档

开场 （5分钟）

  1. 请简要做下自我介绍以及在项目中的角色

PP项目规划 / IPM集成项目管理 / RD需求开发 （20分钟）

  1. 项目计划阶段，是如何进行范围确定以及进行了那些估算？如何进行的？

    SP 1.1 Estimate the Scope of the Project 估算项目范围 PP SG 1 SP 1.2 Establish Estimates of Work Product and Task Attributes 建立对工作产品与任务属性的估算 SP 1.3 Define Project Lifecycle Phases 定义项目生命周期阶段 SP 1.4 Estimate Effort and Cost 估算工作量与成SP 1.1 Elicit Needs 挖掘需要 RD SG 1 SP 1.2 Transform Stakeholder Needs into Customer Requirements 将干系人的需要转换为客户需求

  2. 在估算、计划的过程中利用了哪些组织级的历史数据或历史资料？
  3. 项目计划大致包含哪些方面的内容？请描述一下

    SP 2.1 Establish the Budget and Schedule 建立预算与进度
    SP 2.2 Identify Project Risks识别项目风险
    SP 2.3 Plan Data Management 计划数据管理
    SP 2.4 Plan the Project’s Resources 计划项目资源
    SP 2.5 Plan Needed Knowledge and Skills 计划所需的知识与技能
    项目人员的知识与技能在培训后，是否对人员知识和技能是否满足要求进行跟踪管理？（PMC-SP 1.1)
    SP 2.6 Plan Stakeholder Involvement 计划干系人的参与
    如何选择决策参与人的？（决策计划中未指定、决策分析流程中也未明确选择标准、决策报告中包含了参与人）
    SP 2.4 Perform Make, Buy, or Reuse Analyses 执行自制、购买或复用分析
    SP 2.7 Establish the Project Plan 建立项目计划
    SP 1.1 Establish the Project’s Defined Process 建立项目已定义的过程 IPM SP 1.1
    SP 1.3 Establish the Project’s Work Environment 建立项目工作环境 IPM SP 1.3
    SP 1.6 Establish Teams 建立团队 IPM SP 1.6

  4. 还进行了那些方面的计划？（如：CM计划，测试计划等）
  5. 项目计划是如何活动相关方的同意和获得审批的？

    SP 3.1 Review Plans That Affect the Project 评审影响项目的各项计划
    SP 3.2 Reconcile Work and Resource Levels 协调工作与资源水平
    SP 3.3 Obtain Plan Commitment 获得对计划的承诺

PMC项目监控 / IPM集成项目管理 （20分钟）

  1. 项目实施过程中你对项目进行了哪些跟踪活动？

    SP 1.1 Monitor Project Planning Parameters 监督项目计划参数
    SP 1.2 Monitor Commitments 监督承诺
    SP 1.4 Monitor Data Management 监督数据管理
    SP 1.5 Monitor Stakeholder Involvement 监督干系人的参与
    SP 1.6 Conduct Progress Reviews 进行进展评审
    SP 2.1 Manage Stakeholder Involvement 管理干系人的参与 IPM
    SP 2.3 Resolve Coordination Issues 解决协调问题
    SP 1.5 Manage the Project Using Integrated Plans 使用集成的计划管理项目

  2. 项目在里程碑点进行哪些活动？ PMC SP 1.7

    SP 2.1 Manage Stakeholder Involvement 管理干系人的参与 IPM
    SP 2.3 Resolve Coordination Issues 解决协调问题

  3. 项目的关键依赖是如何进行管理的？
  4. 项目监控过程中发现的问题是如何管理的？

    SP 2.1 Analyze Issues 分析问题
    SP 2.2 Take Corrective Action 采取纠正措施
    SP 2.3 Manage Corrective Actions 管理纠正措施
    SP 2.3 Resolve Coordination Issues 解决协调问题 IPM SP 2.3

  5. 项目在收尾阶段进行哪些活动？如何为组织的过程财富库做贡献？ IPM SP 1.7
    项目负责人如何向组织级风险库中提交风险？

RSKM风险管理（8分钟）

  1. 项目的风险管理是如何进行的？进行了哪些主要风险管理活动？

    SP 2.1 Identify Risks 识别风险
    基于组织风险识别表识别风险外，是否有其它的新识别的风险？
    SP 2.2 Evaluate, Categorize, and Prioritize Risks 评价、分类风险并划分风险优先级
    SP 3.1 Develop Risk Mitigation Plans 制订风险缓解计划
    SP 3.2 Implement Risk Mitigation Plans 实施风险缓解计划
    SP 2.2 Identify Project Risks识别项目风险 PP SP 2.2
    SP 1.3 Monitor Project Risks 监督项目风险 PMC SP 1.3

  2. 公司有没有关于风险识别和管理的指南或策略？ RSKM SG 1

    SP 1.1 Determine Risk Sources and Categories 确定风险来源与类别
    SP 1.2 Define Risk Parameters 定义风险参数
    SP 1.3 Establish a Risk Management Strategy 建立风险管理策略

MA质量 （7分钟）

  1. 项目有哪些度量相关的活动？
  2. 这些度量活动怎样进行计划的？ MA SG 1

    SP 1.1 Establish Measurement Objectives 建立度量目标
    SP 1.2 Specify Measures 明确说明度量项
    SP 1.3 Specify Data Collection and Storage Procedures 明确说明数据收集与存储的规程
    SP 1.4 Specify Analysis Procedures 明确说明分析规程

  3. 你的项目对度量数据做了哪些分析？ MA SG 2

    SP 2.1 Obtain Measurement Data 获得度量数据
    SP 2.2 Analyze Measurement Data 分析度量数据
    SP 2.3 Store Data and Results 存储数据与结果
    SP 2.4 Communicate Results 沟通结果

VER验证（评审和测试） / VAL确认（验收） （10分钟）

  1. 在整个项目阶段，策划了有哪些验证和确认活动？验证时机、验证方法、验证目的、验证对象等。

  • SP 1.1 Select Products for Validation 选择需要确认的产品
    SP 1.2 Establish the Validation Environment 建立确认环境
    SP 1.3 Establish Validation Procedures and Criteria 建立确认规程与准则

  • SP 1.1 Select Work Products for Verification 选择需要验证的工作产品
    SP 1.2 Establish the Verification Environment 建立验证环境
    SP 1.3 Establish Verification Procedures and Criteria 建立验证规程与准则
  1. 同行评审活动的数据会进行分析吗？什么时机分析？从哪些维度分析？能举例说明某一次的分析结果和结论吗？目前项目的同行评审的效率和效果，您怎么看？

GG共通目标
